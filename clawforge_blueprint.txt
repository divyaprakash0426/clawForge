Understanding the battlefield
================================================================================

Stats:

THE CORE INSIGHT: VOLUME != VALUE
ClawHub already has 13,729 skills. Yet security researchers estimate ~20% are
low-quality or risky (ClawHavoc attack, early 2026). The community desperately
needs curated, cross-variant, security-audited skills — not more raw skills.
The gap is the same one that made everything-claude-code (46K+ stars) successful:
being a curated, production-tested system rather than just another entry in an
overcrowded registry.

CLAW VARIANT LANDSCAPE (COMPATIBILITY MATRIX)
Every ClawForge skill is tagged and tested against variants — no existing
collection does this.

PART 2 — INSPIRATION ANALYSIS
What made winners win
================================================================================

EVERYTHING-CLAUDE-CODE (46K+ stars, Anthropic hackathon winner)
Built by Affaan Mustafa at the Cerebral Valley x Anthropic hackathon. Won not by
being a list of configs, but by being a complete system: 13 expert agents, 40+
skills, 32 slash commands, hooks, rules, MCP configs, and a security scanner
(AgentShield with 1,282 tests). Four-layer architecture. Installable as a plugin
marketplace entry in one command. The key was completeness and production-readiness
— not quantity.

ANDREJ KARPATHY SKILLS REPO
Curated skills tailored to one domain expert's workflow. Won on depth and
credibility — each skill was deeply reasoned, not template-generated. Showed that
quality beats quantity, and that a named persona gives a collection authority.

USDC OPENCLAW HACKATHON — CLAWSHIELD (winner)
A plug-and-play security skill that scans repos for unsafe patterns, enforces a
permission manifest, and emits auditable receipts. Won by solving the single most
urgent community pain point: untrusted skill supply chain risk. Judges loved that
it was immediately useful and addressed the ClawHavoc attack aftermath.

CHINA "RAISING LOBSTERS" ECOSYSTEM — KEY SIGNAL
Chinese developers created underground "skill packs" that sold for tens of thousands
of RMB. Tencent launched QClaw with 5,000 prebuilt skills and 3-minute deployment.
The demand pattern is clear: non-developers want batteries-included skill bundles
they can drop in and use immediately, not individual skills they need to vet and
configure. No English-language equivalent exists.

PART 5 — REPOSITORY ARCHITECTURE
Technical blueprint
================================================================================

ROOT DIRECTORY STRUCTURE:

clawforge/
├── README.md              <- install-in-one-command, variant matrix
├── CONTRIBUTING.md        <- skill authoring standards
├── SECURITY.md            <- responsible disclosure + ClawHavoc guidance
├── install.sh             <- interactive setup: detect variant, copy skills
├── install-skill.sh       <- single-skill installer with compat check
├── .github/
│   └── workflows/
│       ├── skill-lint.yml      <- validate SKILL.md frontmatter on PR
│       ├── compat-test.yml     <- test skills against variant matrix
│       └── security-scan.yml  <- run skill-sentinel on all PRs
├── skills/                <- 30 flagship skills
├── souls/                 <- SOUL.md personality configs (onlycrabs.ai)
├── tools/                 <- skill scaffolding CLI
│   └── forge-skill.sh     <- interactive skill generator
└── docs/
    ├── skill-format.md    <- SKILL.md spec + compat tags
    └── variant-guide.md   <- which skills work on which variant

INDIVIDUAL SKILL DIRECTORY (example: arch-sentry):

skills/arch-sentry/
├── SKILL.md          <- YAML frontmatter + runbook instructions
├── README.md         <- human-facing docs, screenshots, use cases
├── install.sh        <- optional one-time setup (pacman deps)
├── scripts/
│   ├── audit.sh      <- core logic scripts
│   └── report.sh
└── COMPAT.md         <- variant compatibility notes

SKILL.MD FRONTMATTER SCHEMA (ClawForge standard):

---
name: arch-sentry
description: Arch Linux system health monitor with chat-driven cleanup approvals.
version: 1.0.0
metadata:
  openclaw:
    emoji: "🦞"
    requires:
      bins: ["pacman", "curl"]
      env: []
    primaryEnv: null
    compat:
      openclaw: full        # full | partial | unsupported
      zeroclaw: full
      picoclaw: partial     # no scheduled heartbeat support
      nullclaw: unsupported # no shell exec
      nanobot: full
      ironclaw: partial     # sandboxed shell
    security_tier: L1       # L1 (read-only) | L2 (writes) | L3 (prod/creds)
    tags: [devops, arch-linux, system, automation]
---

PART 6 — BUILD PLAN FOR GITHUB COPILOT CLI
AI agent build blueprint
================================================================================

Agentic workflow for GitHub Copilot CLI to scaffold and build the entire repo.
Each phase is a discrete prompt chain.

PHASE 1 — Scaffold (Day 1, ~2 hours)

  gh copilot suggest "Create a GitHub repo named clawforge with the directory
  structure described in ARCHITECTURE.md. Initialize with README, CONTRIBUTING,
  SECURITY, install.sh, and GitHub Actions for skill linting (validate SKILL.md
  YAML frontmatter on PR) and security scanning."

  gh copilot suggest "Write an install.sh that: (1) detects which Claw variant is
  installed by checking for openclaw, zeroclaw, picoclaw, nullclaw binaries in
  PATH; (2) shows a compatible skills subset based on detected variant; (3) copies
  selected skills to the correct ~/.openclaw/skills or equivalent directory;
  (4) prints a summary of installed skills."

PHASE 2 — Core skills generation (Day 1-2, ~6 hours)

  gh copilot suggest "Generate a complete SKILL.md for the 'arch-sentry' skill.
  It should: (1) YAML frontmatter with all ClawForge compat fields; (2) a
  runbook-style body with numbered steps for pacman cache audit, orphan detection,
  pacnew conflict resolution; (3) stop conditions (abort if no sudo); (4) output
  format spec; (5) example invocations."

  # Repeat for each skill in the catalog with domain-specific context:
  gh copilot suggest "Generate skill-sentinel/SKILL.md — a pre-install security
  scanner. Steps: (1) fetch SKILL.md from target URL; (2) extract all shell
  command patterns with regex; (3) flag: curl|wget to external IPs, base64
  encoded payloads, credential env var reads, suspicious bin executions;
  (4) output a JSON risk report; (5) block install if risk_score > 7."

PHASE 3 — forge-skill CLI tool (Day 2, ~3 hours)

  gh copilot suggest "Build tools/forge-skill.sh — an interactive Bash CLI that:
  (1) prompts for skill name, description, domain (devops/research/productivity/
  health/security/creative/finance); (2) prompts for required bins and env vars;
  (3) prompts for variant compatibility for each of 6 variants; (4) prompts for
  security tier L1/L2/L3; (5) generates a scaffolded SKILL.md with all frontmatter
  filled and a runbook template body; (6) creates the skill directory structure
  with placeholder README and scripts/."

PHASE 4 — CI/CD pipeline (Day 2, ~2 hours)

  gh copilot suggest "Write .github/workflows/skill-lint.yml. On every PR that
  touches skills/**, run: (1) parse all SKILL.md files with yq; (2) validate
  required frontmatter fields (name, description, version, metadata.openclaw.compat,
  metadata.openclaw.security_tier); (3) check that security_tier L3 skills have a
  human-review label on the PR; (4) post a PR comment with a compatibility matrix
  table showing which variants are supported by changed skills."

PHASE 5 — Demo & submission (Day 3)

  gh copilot suggest "Write a comprehensive README.md for the clawforge repo.
  Include: (1) one-command install; (2) variant compatibility table; (3) skill
  catalog organized by domain with emoji, name, one-liner; (4) security philosophy
  section; (5) contribution guide link; (6) link to accompanying DEV article.
  Tone: developer-friendly, direct, no marketing fluff."

  # Record demo video: install on OpenClaw + NullClaw simultaneously, showing
  # compat check, then run arch-sentry and skill-sentinel live.
